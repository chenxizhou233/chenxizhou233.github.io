Week4

因为外部观测不到 τ\tauτ transition，那么我们能把 τ\tauτ transition 前后的状态合并吗？

当然不行，因为这样有可能把两个完全不同的东西整成 bisimilar 甚至 iso 的，而这与现实明显不符。

Weak Transition

为了描述这些情况，我们引入了 Weak Transition. 本质上就是两个状态间如果有任意多个 τ\tauτ transition + 随便一个 transition α\alphaα (可以为 τ\tauτ ), 那么就称这个 transition 为 τ\tauτ transition。

几个注意点

  • 一个强 τ\tauτ transition 也 implies 一个 Weak τ\tauτ transition.
  • 每个状态都有到自己的 Weak Transition
  • 接上一点：如果要求 Observation Congruence 的话，那么必须要显式的 τ\tauτ transition，阴式的不行。

Weak Bisimulation

定义

类似，对 Weak Transition 我们也可以定义 Bisimulation

对于左边，找 Transition 变更难了（因为得包含 τ\tauτ ），但是右边变简单了。
因此，我们强化左边的条件为 Strong Transition，获得一个新的 Lemma。

这样就得到了一个更加简便的 Weak Bisimulation 判定方法。

一个🌰：

stateDiagram-v2

    Mid1 --> Done1 : τ

    Start0 --> Done0 : a

    Start2 --> Mid2 : τ
    Mid2 --> Done2 : a

这三个东西事 Weak Bisimilar 的。

可以这么理解： 跟 τ\tauτ 有关的都可以和到有真正 transit 前/后 最后一步。

同时 Weak 的定义也让攻击更加简单。因为你其实是可以留在一个状态的，这本身也是一个 τ\tauτ Transition.
有了 Weak，在换边后就有了更多的选择。

性质

这个满足 reflexive， transitive，symmetric

但是不满足 trace preserving

因此我们引入 Weak Trace， 就是把 τ\tauτ 省略掉的 Trace，这样我们就可以说

Weak bisimilar 保留了 trace， 还是 Weakly termination sensitive。

以及它不是 Congruent Relation

这就引出了下一个概念：

Observation Congruence

在 Weak Bisimilar 的基础上更进了一步，认为如果两边经过一个 trace 是 某个Weak Transition 转移后的状态 Weakly Bisimilar，那么这两个 LTS term Observation Congruence。

注意因为这里的 Trace 必须存在一个显式的 Transition。因此在 Weak 最小化的时候必须在第三部重建一些 τ\tauτ 来维护 Observation Congruence。

这个东西满足了我们对等价关系的所有想象。

Weak Minimization

  1. 先按正常的最小化流程，只考虑 Weak Transition （注意每个 state 有到自己的 τ\tauτ transition）
  2. 考察哪些 Transition 删除后还能通过 Weak Transition 重建出来。
  3. 看看每个等价类里面有没有 τ\tauτ transition, 如果有，那么加一个 τ\tauτ 自环 （实际作用见上一部分）

期中复习

这里记录的都是做题相关的信息。

Concurrent的定义

我们认为，一个分时进行的任务是 Concurrent。如果这些分时任务有重叠，那么就是 Parallel 的。感觉上 Concurrent是一个更弱的定义，包含 Parallel。

Concurrent Model 的约定

建模遵循以下原则：

  1. 自动机样式 （State， Transition）。注意每个 LTS 都必须要有 starting state
  2. 状态对于一个 Process 是内部的，无法被外部观测
  3. Transition 是状态转移，都有一个 Action 名字
  4. Act=Int∪ComAct = Int \cup ComAct=Int∪Com
  5. 状态转移瞬间完成，时间只在 State 内流动。

进一步，关于时间，我们有以下 Core Assumptions:

  • Concurrent processes 可以任意相对速度执行。
  • 从时间角度，我们最多只考虑 Process 之间的相互顺序。

Labeled Transition System (LTS)

这个就是前面说的建模。类似自动机，定义在结构
